# Contributing to Pier

Pier's CLI is closed-source, but this repository is where the community engages with the product. We welcome bug reports, feature requests, and documentation feedback.

## Issues first

The most valuable contributions are well-described issues:

- **Bug reports** — use the bug report template. Include your version (`pier --version`), OS, terminal, and clear steps to reproduce.
- **Feature requests** — describe the problem you're trying to solve, not just the solution you have in mind.
- **Documentation** — spotted something wrong or unclear in `docs/`? Open an issue or a small pull request.

Search [existing issues](https://github.com/alphabench/pier/issues) before filing a new one.

## Pull requests

This repo contains docs, examples, and install tooling — not product source. We accept pull requests against those:

- Keep changes focused and atomic.
- Update relevant docs when behavior changes.
- Never include secrets, credentials, or internal endpoints.

Larger changes are best discussed in an issue first so we can align before you invest time.

## Community values

Be kind, be inclusive, and assume good intent. We're here to teach and to learn. See our [Code of Conduct](./CODE_OF_CONDUCT.md).

## Security

Do not report security vulnerabilities through public issues or pull requests. See [SECURITY.md](./SECURITY.md).
