# Security Policy

Thank you for helping us keep Pier secure.

## Reporting a vulnerability

Please report security vulnerabilities **privately** by emailing [hello@piercode.com](mailto:hello@piercode.com). Do **not** open a public GitHub issue for security problems.

When reporting, include where possible:

- A description of the vulnerability and its impact.
- Steps to reproduce, or a proof of concept.
- The Pier version (`pier --version`) and your operating system.

We will acknowledge your report, investigate, and keep you updated on the fix. Please give us a reasonable window to address the issue before any public disclosure.

## Scope

This policy covers the Pier CLI and its distribution channel (`dl.piercode.com`). For account, billing, or website issues, the same email applies.
